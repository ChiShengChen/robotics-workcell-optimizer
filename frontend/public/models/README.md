# /public/models/ — robot mesh slot

The 3D preview tries to load a static GLB mesh for the active robot **on top of**
the procedural arm whenever a file is available here. If no file exists the
procedural fallback is used silently (no error in the UI).

## Filename convention

For a robot whose `model` id in `backend/app/data/robots.json` is `M-410iC/110`,
drop a file at:

```
frontend/public/models/m_410ic_110.glb
```

The slug is generated by `frontend/src/lib/meshes.ts::modelSlug`:

```ts
modelSlug("M-410iC/110") === "m_410ic_110"
```

(Lowercase, non-alphanumerics → `_`, leading/trailing `_` stripped.)

## Optional manifest override

If you don't want to follow the slug convention, drop a `manifest.json` here:

```json
{
  "robots": {
    "M-410iC/110": "/models/fanuc_m410ic_arm_only.glb",
    "IRB 460":    "/models/abb_irb460.glb"
  },
  "pallets": {}
}
```

`lib/meshes.ts::loadMeshManifest` will fetch this on first lookup; an explicit
mapping wins over the slug convention.

## Format requirements

- **Format**: GLB (binary glTF). `.gltf` + bin + textures works too but is more
  fragile.
- **Up axis**: `+Y` (the Three.js convention drei expects). If your source is
  `+Z` (most CAD), re-export from Blender with `+Y` up.
- **Origin**: at the robot's **base centre on the floor**. The component
  positions the mesh at `(robot.x_mm * 0.001, 0, robot.y_mm * 0.001)`, no
  vertical offset.
- **Units**: metres. CAD/STEP exports usually default to mm — scale by 0.001
  on export.
- **Static pose**: the GLB does NOT need to be rigged. The procedural arm
  still drives the IK animation underneath; the GLB is a cosmetic overlay
  for the base / cabinet / housing detail. If you provide a fully-articulated
  arm GLB, you'll see it overlay our IK arm — usually you'd want to either
  delete the procedural arm or trim the GLB to just the base + lower frame.
- **Polygon budget**: aim for < 50k tris per GLB. We don't bake LODs.

## How to convert URDF / Collada → GLB

```bash
# Blender ≥ 4.0
blender --background --python-expr "
import bpy
bpy.ops.import_scene.collada(filepath='input.dae')
# Bake transforms / merge meshes if needed
bpy.ops.export_scene.gltf(
    filepath='output.glb',
    export_format='GLB',
    export_apply=True,
    export_yup=True,
)
"
```

For URDF specifically:
```bash
pip install urchin
python -c "
import urchin
robot = urchin.URDF.load('robot.urdf')
robot.show()  # interactive preview, then save scene as GLB via three or trimesh
"
```

## Verifying it loaded

Open the browser DevTools Network tab. When you switch to the 3D preview tab
you should see `/models/<slug>.glb` returning 200 (not 404). The mesh appears
in the next render frame.
